# Markdown 优化指令模板

## 使用场景

当需要对 Markdown 文章进行公众号排版时，使用以下指令。

## 核心原则

```
WARNING: 绝对不能修改任何具体的文字内容！
WARNING: 只能修改格式（标题层级、加粗、分隔线）
WARNING: 保持原文的每一个字、每一句话不变
WARNING: 不要添加任何 [DATA]、[INSIGHT] 等标签
```

**可修改的格式**：
- 删除 bullet points（●、-、*）
- 删除或简化破折号
- 添加标题层级
- 用 `---` 分隔章节
- 保留原文，不添加额外标签

**设计目标**：
- 清晰的视觉层级
- 干净的段落排版
- 适度的重点强调
- 简洁克制的整体风格

## 排版要求

### 1. 标题结构

根据内容逻辑添加标题层级：

- 一级标题（#）：文章总标题
- 二级标题（##）：主要章节
- 三级标题（###）：章节内小节

### 2. 重点突出

识别关键信息，用 **加粗** 标记：

- 关键数据：具体数字、百分比
- 核心观点：主要结论
- 重要概念：专业术语
- 关键名称：公司、产品名

**规则**：
- 不要过度使用加粗
- 每个段落 1-2 处重点即可
- 加粗核心词语，而非整句

### 3. 章节分隔

使用 `---` 分隔主要章节：

- 在主题切换处添加
- 不要过于频繁
- 每个主要部分之间使用

### 4. 格式符号处理

| 原文格式 | 处理方式 |
|---------|---------|
| `●关键事实：` | 删除 `●`，保留文字作为普通段落 |
| `●洞察：` | 删除 `●`，保留文字作为普通段落 |
| `1% - xxx` | 可保留作为标题 |
| `- xxx` 列表 | 可转换为普通段落 |

## 禁止事项

- 修改原文的任何文字表述
- 改变原句的意思或措辞
- 添加原文没有的观点
- 过度使用加粗
- 为内容添加特殊样式标记

## 输出格式

只输出优化后的 Markdown 内容，不要添加任何解释说明。

## 示例

### 示例 1

**原文**：
```
●关键事实：智谱与MiniMax为代表的的中国头部大模型公司在研发投入、收入甚至未来估值上，大约只有美国同行（如OpenAI）的1%。

●洞察：喜忧参半，我们用美国1%的钱，做到了对方80%甚至90%的模型能力。
```

**优化后**：
```
## 1% - 中美AI企业的"百分之一"效应

智谱与MiniMax为代表的的中国头部大模型公司在研发投入、收入甚至未来估值上，大约只有美国同行（如OpenAI）的**1%**。

喜忧参半，我们用美国1%的钱，做到了对方**80%甚至90%**的模型能力。
```

### 示例 2

**原文**：
```
根据招股书研发费用中人员工资测算，MiniMax与智谱研发人员平均成本约为90万元。

相比硅谷动辄百万美金的年薪，中国AI人才的平均水位仍处于合理阶段。这组数据一部分程度上解释了中国大模型公司为何能靠几亿美金支撑至今。
```

**优化后**：
```
## 90万 - 人才红利背后的价格差

根据招股书研发费用中人员工资测算，MiniMax与智谱研发人员平均成本约为**90万元**。

相比硅谷动辄百万美金的年薪，中国AI人才的平均水位仍处于合理阶段。这组数据一部分程度上解释了中国大模型公司为何能靠几亿美金支撑至今。
```

## 检查清单

完成优化后检查：

- [ ] 所有原始文字内容都保留了
- [ ] 删除了冗余的格式符号（●、- 等）
- [ ] 没有添加任何 [DATA]、[INSIGHT] 等标签
- [ ] 标题层级清晰，逻辑合理
- [ ] 重点内容适度加粗
- [ ] 章节之间有适当的分隔
- [ ] 没有修改原文的任何具体文字表述
